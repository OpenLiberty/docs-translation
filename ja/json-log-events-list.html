JSON ログ・イベントの参照リスト

Open Liberty は、サーバー・ランタイムおよびアプリケーションから JSON ロギング・イベントを生成します。 これらのイベントを使用して、アプリケーションの動作をより良く理解するのに役立つデータを収集および分析することができます。

以下のタイプのイベントが Open Libertyによって生成されます。

  • メッセージ・イベント

  • トレース・イベント

  • FFDC イベント

  • HTTP アクセス・イベント

  • 監査イベント

メッセージ・イベント

以下の表に、メッセージ・イベントのフィールドと、各フィールドの説明を示します。

メッセージ・イベント・フィールド
フィールド 説明

タイプ

イベント・タイプを識別するストリング。

ホスト

イベントのソースとなっているサーバーのホスト名。

ibm_userDir

イベントのソースとなっているサーバーのユーザー・ディレクトリー。

ibm_serverName

イベントのソースとなっているサーバーの名前。

メッセージ

メッセージ ID で始まる、ログ・レコードからのメッセージ。

ibm_threadId

ログ行内のスレッド ID (00000015 など)。 スレッド ID は、数値ではなく、ストリングです。

ibm_datetime

イベントが発生した時刻。

ibm_messageId

ログ行内のメッセージ ID。これを使用して、特定タイプのエラーを検索することができます (SRVE0250I など)。

モジュール

ログ・レコードからのロガー名。

loglevel

重大度標識 (F = 致命的、E = エラー、W = 警告、A = 監査、I = 情報、O = SystemOut、R = SystemErr)。

ibm_methodName

ログ・レコードからのメソッド名。

ibm_className

ログ・レコードからのクラス名。

ibm_sequence

イベントのシーケンス番号。これは、同じタイム・スタンプを持つレコードをソートする場合に役立ちます。

EXT_THREAD

イベントのソースとなっているスレッドの名前。

ext_appName

メッセージをログに記録したアプリケーションの名前。

サポートされる監査イベントとその監査データ

Open Liberty Audit フィーチャーは、サーバー・ランタイム環境およびアプリケーションからのセキュリティー詳細を含む監査可能イベントをキャプチャーします。 監査イベントから生成されたデータを使用して、構成された環境を分析することができます。

Open Liberty は、JSON 形式または CADF 形式のいずれかで監査イベントを生成できます。 監査イベントは、以下の JSON 形式のタイプでキャプチャーされ、構成された環境を改善できるさまざまな領域を識別するのに役立ちます。

  • 監査サービスの管理 (SECURITY_AUDIT_MGMT)

  • SCIM 操作/メンバー管理 (SECURITY_MEMBER_MGMT)

  • Servlet 3.0 API: ログイン/認証 (SECURITY_API_AUTHN)

  • Servlet 3.0 API: ログアウト (SECURITY_API_AUTHN_TERMINATE)

  • フォーム・ログアウト (SECURITY_AUTHN_TERMINATE)

  • 基本認証 (SECURITY_AUTHN)

  • クライアント証明書認証 (SECURITY_AUTHN)

  • フォーム・ログイン認証 (SECURITY_AUTHN)

  • サーブレット runAs 委任 (SECURITY_AUTHN_DELEGATION)

  • EJB 委任 (SECURITY_AUTHN_DELEGATION)

  • 基本認証へのフェイルオーバー (SECURITY_AUTHN_FAILOVER)

  • 無保護サーブレット許可 (SECURITY_AUTHZ)

  • JACC Web 許可 (SECURITY_AUTHZ)

  • JACC EJB 許可 (SECURITY_AUTHZ)

  • EJB 許可 (SECURITY_AUTHZ)

  • JMS 認証 (SECURITY_JMS_AUTHN)

  • JMS 許可 (SECURITY_JMS_AUTHZ)

  • SAF 許可サービス API 要求 (SECURITY_SAF_AUTHZ)

  • SAF 許可例外 (SECURITY_SAF_AUTHZ_DETAILS)

  • JMX MBean 登録 (JMX_MBEAN_REGISTER)

  • JMX MBean 操作 (JXM_MBEAN)

  • JMX MBean 属性操作 (JMX_MBEAN_ATTRIBUTES)

  • JMX 通知 (JMX_NOTIFICATION)
