メトリック参照リスト

メトリック参照表には、 Open Libertyで使用可能なすべてのメトリックがリストされ、説明されています。 メトリック・データを使用して、マイクロサービス・システムの状況を効果的にモニターします。

メトリックは、アプリケーション、 Open Liberty ランタイム、および Java 仮想マシン (JVM) から取得できます。 これらは、 Prometheusなどのデータベース・ツールで収集して保管し、 Grafanaなどのダッシュボードに表示することができます。 アプリケーションへのプログラム識別情報の組み込みについて詳しくは、 Microservice 可観測性とメトリックを参照してください。 サード・パーティーのモニター・システムにメトリック・データを送信するために MicroProfile Metrics 5.0 と Micrometer を統合する方法について詳しくは、 MicroProfile Metrics を使用した独自のモニター・ツールの選択を参照してください。

基本メトリックおよびベンダー・メトリック

MicroProfile Metrics フィーチャーが有効になっている場合、基本メトリックのセットが常に報告されます。 Open Liberty サーバー内のさまざまなモニター・コンポーネントから使用可能なベンダー・メトリックを収集することで、このメトリック・セットを拡張できます。 パフォーマンス・モニター と MicroProfile メトリック・フィーチャーの両方を有効にすると、 /metrics エンドポイントでのベンダー・メトリックのレポート作成が有効になります。 パフォーマンス・モニター機能は、使用可能なすべてのモニター・コンポーネントから統計データを取得します。 MicroProfile Metrics 2.3 で導入された REST 基本メトリックも、パフォーマンス・モニター・フィーチャーに依存しています。 MicroProfile Metrics 2.3より前は、パフォーマンス・モニター・フィーチャーを明示的に構成する必要がありました。 MicroProfile Metrics 2.3以降では、Performance Monitoring フィーチャーは、始動時に MicroProfile Metrics によって自動的に使用可能になります。

必要なデータのみを収集するようにメトリックをフィルタリングします

デフォルトでは、すべてのモニター・コンポーネントが有効になっています。 サーバーが必要以上のメトリック・データを収集している場合は、使用する予定のベンダー・メトリックのみを収集することで、サーバーのパフォーマンスを向上させることができます。 報告するベンダー・メトリックのサブセットのみを構成するには、 server.xml ファイル内の monitor 構成エレメントの filter 属性で、モニターするコンポーネントを指定します。 各ベンダー・メトリックの関連するモニター・コンポーネントを識別するには、メトリック参照表の 「モニター・コンポーネント」 列を参照します。

MicroProfile Metrics で使用されるモニター・コンポーネントのみを有効にするには、以下のコードを server.xml ファイルに追加します。

<monitor  filter="ConnectionPool,ThreadPool,RequestTiming,Session,WebContainer,REST,GrpcClient,GrpcServer"/>

すべてのベンダー・メトリックを無効にするが、 REST 基本メトリックを保持するには、以下の例に示すように server.xmlを構成します。

<monitor  filter="REST"/>

すべてのモニター・コンポーネントを無効にするには、以下のコードを server.xml ファイルに追加します。

<monitor  filter=" "/>

メトリックのリファレンス・テーブル

以下の 2 つの表は、 MicroProfile Metrics 5.0 または MicroProfile Metrics 4.0 (またはそれ以前のバージョン)を使用する場合に使用可能なメトリックをリストしています。 どちらの表でも、各メトリックの後に Prometheus メトリック名がリストされています。 表には、 Open Libertyで使用可能なすべてのメトリックのメトリック・タイプ、メトリック単位、および説明もリストされています。 ベンダー・メトリックの場合、メトリックのフィルタリングに使用できる関連モニター・コンポーネントも含まれます。 表の 「必要なフィーチャー」 列には、そのメトリック・データを取得するために有効にする必要があるフィーチャーが含まれています。 「導入されたバージョン」 列は、メトリックを収集するために有効にする必要がある機能の最小バージョンを指定します。

メトリック参照表内の単位

どちらの表でも、メトリック単位は、メトリック名と Prometheus 名の両方とともに含まれています。 MicroProfile 4.0 以前では、 Prometheus エクスポーターがメトリックをベース・ユニットにスケーリングするため、これらのユニットは異なる可能性があります。 例えば、 ft.<name>.bulkhead.executionDuration メトリックはナノ秒単位で内部的に記録されますが、 Prometheus の出力は秒単位です。 MicroProfile Metrics 5.0では、メトリックに関連付けられている単位は、 Prometheus 出力に反映される単位です。 メトリックは基本単位にスケーリングされません。

メトリック名の横に単位がリストされていない場合、そのメトリックに関連付けられている単位はありません。
