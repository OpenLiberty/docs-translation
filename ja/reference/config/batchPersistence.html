  • MatchCurrentState

  • MatchOriginalRequest

MatchOriginalRequest

接続が共有のためにどのようにマッチングされるかを指定します。
MatchCurrent状態
接続を共有する場合、接続の現在の状態に基づいて突き合わせが行われます。 サーバーの稼働中に行われた更新は、トランザクションのそれぞれの最初の接続ハンドルで適用されます。
MatchOriginal要求
接続を共有する場合、元の接続要求に基づいて突き合わせを行います。

containerAuthDataRef

最上位の authData エレメント (ストリング) の参照。

res-auth=CONTAINER が指定されたリソース参照の認証別名がバインディングで指定されていない場合に適用される、コンテナー管理認証のデフォルトの認証データ。 サーバーの稼働中に更新されると、変更は新規の接続要求に適用されます。使用中の接続は影響されません。

isolationLevel

  • TRANSACTION_NONE

  • TRANSACTION_READ_COMMITTED

  • TRANSACTION_READ_UNCOMMITTED

  • TRANSACTION_REPEATABLE_READ

  • TRANSACTION_SERIALIZABLE

  • TRANSACTION_SNAPSHOT

デフォルトのトランザクション分離レベル。 これが未指定で、データベースが DB2、Derby、Informix、Microsoft SQL Server または Sybase と特定される場合、TRANSACTION_REPEATABLE_READ が使用されます。 他のデータベースで未指定の場合、TRANSACTION_READ_COMMITTED が使用されます。 サーバーの稼働中に行われた更新は、新規の接続要求に適用されます。使用中の接続は影響されません。
TRANSACTION_NONE
JDBC ドライバーがトランザクションをサポートしないことを示します。
TRANSACTION_READ_COMMITTED
ダーティー読み取りは防止されます。反復不能読み取りおよび幻像読み取りが発生する可能性があります。
TRANSACTION_READ_UNCOMMITTED
ダーティー読み取り、反復不能読み取り、および幻像読み取りが発生する可能性があります。
TRANSACTION_REPEATABLE_READ
ダーティー読み取りおよび反復不能読み取りは防止されます。幻像読み取りが発生する可能性があります。
TRANSACTION_SERIALIZABLE
ダーティー読み取り、反復不能読み取り、および幻像読み取りは防止されます。
TRANSACTION_SNAPSHOT
Microsoft SQL Server JDBC ドライバーおよび DataDirect Connect for JDBC ドライバーのスナップショット分離。
