审计日志事件引用列表 (CADF 格式)

Open Liberty Audit 功能部件可捕获包含来自服务器运行时环境和应用程序的安全详细信息的可审计事件。 您可以使用从审计事件生成的数据来分析配置的环境。

Open Liberty 可以生成 CADF 或 JSON 格式的审计事件。 可以通过以下 CADF 格式类型捕获审计事件，以帮助确定可以改进所配置环境的不同区域:

  • 管理审计服务 (SECURITY_AUDIT_MGMT)

  • SCIM 操作/成员管理 (SECURITY_MEMBER_MGMT)

  • Servlet 3.0 API: 登录/认证 (SECURITY_API_AUTHN)

  • Servlet 3.0 API: 注销 (SECURITY_API_AUTHN_TERMINATE)

  • 表单注销 (SECURITY_AUTHN_TERMINATE)

  • 基本认证 (SECURITY_AUTHN)

  • 客户机证书认证 (SECURITY_AUTHN)

  • 表单登录认证 (SECURITY_AUTHN)

  • Servlet runAs 授权 (SECURITY_AUTHN_委派)

  • EJB 委派 (SECURITY_AUTHN_委派)

  • 故障转移到基本认证 (SECURITY_AUTHN_FAILOVER)

  • 不受保护的 servlet 授权 (SECURITY_AUTHZ)

  • JACC Web 授权 (SECURITY_AUTHZ)

  • JACC EJB 授权 (SECURITY_AUTHZ)

  • EJB 授权 (SECURITY_AUTHZ)

  • JMS 认证 (SECURITY_JMS_AUTHN)

  • JMS 授权 (SECURITY_JMS_AUTHZ)

  • SAF 授权服务 API 请求 (SECURITY_SAF_AUTHZ)

  • SAF 授权异常 (SECURITY_SAF_AUTHZ_DETAILS)

  • JMX MBean 注册 (JMX_MBEAN_REGISTER)

  • JMX MBean 操作 (JMX_MBEAN)

  • JMX MBean 属性操作 (JMX_MBEAN_ATTRIBUTES)

  • JMX 通知 (JMX_NOTIFICATION)

您可以使用 SECURITY_AUDIT_MGMT 事件从审计服务的管理中捕获审计信息。 下表提供了 SECURITY_AUDIT_MGMT 事件的字段以及每个字段的描述:

字段 描述

eventName

审计事件的名称。

eventSequence数字

审计事件的序号。

eventTime

事件发生的时间。

observer.id

事件的观察者的标识。

observer.name

事件的观察者的名称: AuditService (对于审计服务); AuditHandler: name of handler implementation (对于处理程序启动)。

observer.typeURI

事件观察者的唯一 URI: service/server。

outcome

活动的结果。

target.id

操作目标的标识。

target.typeURI

事件目标的唯一 URI: server/audit/start (对于 AuditService 或处理程序启动); server/audit/stop (对于 AuditService 或处理程序停止)。
