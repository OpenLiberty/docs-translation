使用管理中心管理Open Liberty

管理中心是Open Liberty服务器的网络图形用户界面（GUI）。 您可以使用管理中心管理服务器配置、OpenIDConnect 权限和 Java 批量作业。

设置管理中心

  1. 如果运行时尚未安装管理中心功能，请在 "path_to_liberty/wlp/bin目录下运行featureUtility installFeature命令进行安装，如下例所示。

    featureUtility installFeature adminCenter-1.0

  2. 在 "server.xml文件中启用管理中心功能，并为阅读器或管理员配置管理角色。

  3. Admin Center 只能通过 HTTPS 访问，因此必须指定 HTTPS 端口并配置密钥库，以保护包含服务器身份验证凭据的 keystore 文件。 您可以使用下面的示例 "server.xml文件来设置 Admin Center 的测试实例。

本例启用了 "adminCenter-1.0功能，并配置了一个基本用户注册表，定义了 "reader-role和 "administrator-role管理角色的用户。 它还在 httpEndpoint 元素中指定了 HTTPS 端口，并在 keyStore 元素中指定了默认密钥存储和密码引用。

<server description="Admin Center example server">
   <featureManager>
      <feature>adminCenter-1.0</feature>
   </featureManager>
   <basicRegistry id="basic">
      <user name="admin" password="adminpwd" />
      <user name="reader" password="readerpwd" />
   </basicRegistry>
   <administrator-role>
      <user>admin</user>
   </administrator-role>
   <reader-role>
      <user>reader</user>
   </reader-role>
   <keyStore id="defaultKeyStore" password="Liberty" />
   <httpEndpoint id="defaultHttpEndpoint"
                 host="*"
                 httpPort="9080"
                 httpsPort="9443" />
</server>

登录管理中心

在访问管理中心图形用户界面之前，请确保服务器正在运行。 在网络浏览器中，通过以下 URL 进入管理中心登录页面。

https://host_name:port_number/adminCenter

对于 "host_name值，请指定Open Liberty服务器所运行计算机的 IP 地址或域名服务器 (DNS) 主机名。 如果浏览器与服务器运行在同一台计算机上，则可以指定 "localhost作为主机名。 对于 "port_number值，请从 "server.xml文件中的 "httpEndpoint元素指定 "httpsPort值。

例如，如果浏览器与服务器运行在同一台计算机上，并使用前面的 server.xml 配置示例，则可通过以下 URL 访问管理中心。

https://localhost:9443/adminCenter

只有直接从浏览器访问时，才支持访问管理中心。 不支持通过 HTTP 代理服务器访问。

如果您的浏览器提示您确认连接是否可信，请指定例外，或允许连接继续到管理中心。 要登录管理中心，请从管理角色配置中指定用户名和密码。 例如，如果使用之前提供的 "server.xml配置示例，则应指定 "admin为用户名，"adminpwd为密码。

登录管理中心后，浏览器会显示工具箱，其中包含服务器配置和探索工具等工具，以及指向 Open Liberty网站的书签。

下面的屏幕截图显示了管理中心工具箱。

使用服务器配置工具管理和测试服务器配置

您可以使用服务器配置工具查看和编辑Open Liberty服务器配置文件，并测试服务器与不同资源的连接。 要编辑配置文件或测试连接，必须使用 "administrator-role管理角色。

在编辑配置文件之前，工具会提示您在 "server.xml文件中添加remoteFileAccess）元素。 您可以从浏览器窗口复制示例元素，也可以手动进行配置。 如果不添加此元素，配置文件将以只读模式显示。

要启用对配置文件的写入访问，请在 "server.xml文件中添加以下代码。

<remoteFileAccess>
   <writeDir>${server.config.dir}</writeDir>
</remoteFileAccess>

服务器配置工具以两种模式显示配置文件，例如 server.xml 文件 ：设计和源文件。 每种模式都可以通过工具内的浏览器选项卡进行访问。 单击 "设置"图标可自定义这些模式。 例如，在"设计 "模式下可以添加或删除参数说明，在 "源码"模式下可以添加或删除行号。

设计模式通过图形化控件和内联文档显示配置文件的内容。 在此模式下，选择配置中的元素，查看已启用的功能和参数设置。

源文件模式允许直接访问编辑文件文本，并为文件元素提供工具提示说明。 将鼠标悬停在元素和参数上可查看其文档。 可按 Ctrl + Space 键以使用内容辅助，这可以帮助您添加新元素、参数和值。 不过，源码模式编辑器不会验证您指定的配置值。 在源模式下所做的更改会应用到运行中的服务器。 指定无效的配置值可能会产生错误或导致服务器故障。

使用服务器配置工具测试服务器与资源的连接

您可以使用服务器配置工具测试与 Java Database Connectivity (JDBC)、Java EEConnector Architecture(JCA)、Java Message Service(JMS) 和Cloudant资源的连接。

  1. 要使用服务器配置工具测试连接，除了管理员中心功能外，还必须在 "server.xml文件中启用管理员 REST 连接器和MicroProfile OpenAPI功能。 您还必须启用任何功能，以支持正在检查连接的服务器资源。 下面的示例还启用了JDBC功能，以支持数据库连接。

    <featureManager>
   <feature>adminCenter-1.0</feature>
   <feature>restConnector-2.0</feature>
   <feature>mpOpenApi-3.0</feature>
   <feature>jdbc-4.3</feature>
   ...
</featureManager>

  2. 在服务器配置工具中，从设计选项卡的服务器菜单中选择要测试连接的资源。

  3. 在所选资源的显示窗口中，单击测试按钮。

    下面的屏幕截图显示了服务器配置工具中的 "设计"选项卡和所选资源的显示窗口。

  4. 选择要用于连接测试的身份验证方法。 三种身份验证方法是容器身份验证、应用程序身份验证或无资源引用。

    下面的屏幕截图显示了连接测试页面和身份验证方法。

  5. 在选定的身份验证方法中，指定所需字段，然后单击连接测试按钮。 显示测试结果。

管理中心使用 REST API 验证连接。 有关更多信息，请参阅验证服务器连接。

使用探索工具监控和管理资源

您可以使用探索工具监控和管理Open Liberty资源。 该工具提供了查看服务器及其应用程序信息的选项。 您还可以用它来停止、启动或重启资源。

如需在 “探索”工具中更及时地更新服务器和应用程序状态，请将 Web Socket 功能添加到您的服务器配置中。 此功能可实时显示拓扑结构，不受大小限制。 如果没有 Web Socket 功能，管理中心会定期频繁地轮询更改。

您可以从导航栏中选择 "应用程序"视图来监控、停止、启动或重启服务器上运行的应用程序。

导航栏上的监控器视图以图表形式显示服务器和系统指标。 您可以使用监控器视图跟踪堆内存、加载的类、活动 Java 虚拟机 (JVM) 线程和中央处理器 (CPU) 的使用情况。

下面的屏幕截图显示了探索工具的监控视图。

选择 "编辑"图标并从菜单中选择一个选项，即可在 "监控器"视图中添加更多资源指标。 启用性能监控功能后，该菜单可提供更多资源指标。 根据为服务器配置的资源，可能会提供以下资源指标：

  • 活动会话数

  • 活动 Liberty 线程

  • 平均响应时间

  • 平均等待时间

  • 请求计数

  • 已用连接数

使用 Java 批次工具管理 Java 批次作业

如果配置了批次管理功能，就可以访问Java 批次工具。 使用该工具，您可以查看 Java 批次作业的进度和状态、管理其实例并查看其日志文件。

下面的屏幕截图显示了Java 批次工具。

每个批处理作业都有一个 "行动图标，您可以选择它来停止、重启或清除作业；还有一个 "查看日志图标，您可以选择它来查看作业日志。 要查看批次任务的详细信息和指标，请将鼠标悬停在批次任务名称上，然后从工具提示菜单中选择一个选项。

如果批处理作业或作业日志位于远程服务器上，请在每个远程服务器上配置跨源区域共享 (CORS)。 CORS 使 Admin Center 能够从远程服务器请求作业信息。

批次管理功能需要自定义授权才能查看和管理批次作业。 要使用 Java 批次工具，除了访问管理中心所需的阅读器或管理员管理角色外，还必须配置 "com.ibm.ws.batch自定义授权角色。 下面的 "server.xml文件示例显示了 "wanda用户的配置，该用户被授予管理员管理角色和batchAdmin自定义授权角色。

  <administrator-role>
      <user>wanda</user>
  </administrator-role>

  <authorization-roles id="com.ibm.ws.batch">
      <security-role name="batchAdmin">
          <user name="wanda"/>
      </security-role>
  </authorization-roles>

通过此配置，用户有权查看和管理任何已配置的 Java 批量作业。

com.ibm.ws.batch自定义授权角色也可与阅读器管理角色相结合。 这种组合仍允许完全访问 Java 批次工具，但对其他管理中心资源提供只读访问。

但是，如果没有配置自定义批处理授权角色，即使是管理员管理角色的用户也无法查看或管理 Java 批处理作业。
