日志记录 (logging)

控制日志和跟踪消息的捕获和输出。

名称 类型 缺省值 描述

appsWriteJSON

布尔型

否

当消息日志或控制台日志采用 JSON 格式时，允许应用程序将 JSON 格式的消息写至这些目标，不进行修改。

consoleFormat

  • DEV

  • JSON

  • SIMPLE

  • 待定

DEV

控制台的必需格式。 有效值为 DEV、SIMPLE 或 JSON 格式。 缺省情况下，consoleFormat 设置为 WLP_LOGGING_CONSOLE_FORMAT 环境变量（如果已设置）或者 DEV。 避免麻烦：请使用 WLP_LOGGING_CONSOLE_FORMAT 环境变量或 com.ibm.ws.logging.console.format 属性，但启动之后尝试动态更改配置的情况除外。
DEV
使用 dev 日志记录格式。
JSON
使用 JSON 日志记录格式。
SIMPLE
使用简单日志记录格式。
TBASIC
使用 tbasic 日志记录格式。

consoleLogLevel

  • AUDIT

  • 错误

  • INFO

  • OFF

  • WARNING

AUDIT

用于对写入系统流的消息进行过滤的记录级别。 有效值为 INFO、AUDIT、WARNING、ERROR 和 OFF。 缺省情况下，consoleLogLevel 设置为 WLP_LOGGING_CONSOLE_LOGLEVEL 环境变量（如果已设置此环境变量）或 AUDIT。 注意：更改此值之前，请参阅“开发者工具已知限制”主题中的“在修改控制台记录级别设置之后无法与 Liberty 服务器交互”一节中的信息。 避免麻烦：请使用 WLP_LOGGING_CONSOLE_LOGLEVEL 环境变量或 com.ibm.ws.logging.console.level 属性，但启动之后尝试动态更改配置的情况除外。
AUDIT
审计和警告消息将写入系统输出流。 会将错误消息写入系统错误流。
错误
错误消息将写入系统错误流。
INFO
信息，审计和警告消息将写入系统输出流。 会将错误消息写入系统错误流。
OFF
不会将服务器输出写入系统流。 仅会将 JVM 输出写入系统流。
警告
将向系统输出流写入警告消息。 会将错误消息写入系统错误流。

consoleSource

字符串

消息

路由至控制台/console.log 的源的逗号分隔列表。 仅当 consoleFormat=json 时，此属性才适用。 有效值为 message、trace、accessLog、ffdc 和 audit。 缺省情况下，consoleSource 设置为环境变量 WLP_LOGGING_CONSOLE_SOURCE（如果已设置此环境变量）或 message。 注意：要使用审计源，请启用 Liberty 审计功能部件。 通过为 httpEndpoint 设置 accessLogging 元素来启用访问日志。 避免麻烦：请使用 WLP_LOGGING_CONSOLE_SOURCE 环境变量或 com.ibm.ws.logging.console.source 属性，但启动之后尝试动态更改配置的情况除外。

copySystemStreams

布尔型

是

如果为 true，那么写至 System.out 流和 System.err 流的消息将复制到 console.log。 如果为 false，那么这些消息将写至已配置的日志（例如 messages.log 或 trace.log），但不会复制到 console.log。 缺省值为 true。 避免麻烦：请使用 com.ibm.ws.logging.copy.system.streams 属性，但是启动之后尝试动态更改配置的情况除外。

hideMessage

字符串

配置为在 console.log 和 messages.log 文件中隐藏的消息的逗号分隔列表。 如果这些消息配置为隐藏，那么它们会重定向至 trace.log 文件。 避免麻烦：请使用 com.ibm.ws.logging.hideMessage 属性，但是启动之后尝试动态更改配置的情况除外。

isoDateFormat

布尔型

否

日期和时间使用特定于语言环境的格式或 ISO-8601 格式。 可以对该属性或等价属性的值指定 true 或 false。 缺省值为 false。 避免麻烦：请使用 com.ibm.ws.logging.isoDateFormat 属性，但启动之后尝试动态更改配置的情况除外。 如果指定 true 值，那么将在 messages.log 文件、trace.log 文件和 FFDC 日志中使用 ISO-8601 格式。 此格式为 yyyy-MM-dd'T'HH:mm:ss.SSSZ。 如果指定 false 值，那么将根据系统中设置的缺省语言环境来设置日期和时间的格式。 如果找不到缺省语言环境，那么此格式为 dd/MMM/yyyy HH:mm:ss:SSS z。

jsonAccessLogFields

  • 缺省值

  • logFormat

缺省值

当日志采用 JSON 格式时，请使用此属性，在使用 accessLogging logFormat 属性中指定的访问日志字段与使用缺省访问日志字段之间进行选择。
缺省值
使用缺省访问日志字段集。
logFormat
使用与 logFormat匹配的访问日志字段集。

jsonField映射

字符串

当日志采用 JSON 格式时，请使用此属性将缺省字段名称替换为新的字段名称，或者从日志中省略这些字段。 要替换字段名称，请使用以下格式来配置新的字段名称：defaultFieldName:newFieldName?。 对于与指定源的日志关联的字段名称，请使用以下格式: [source:]?defaultFieldName:newFieldName? 其中 [source] 是要指定的源，例如消息，跟踪或 accessLog。 要从日志中省略字段，请指定没有替代项的字段名称，如以下示例中所示：defaultFieldName:。 要重命名或者省略多个字段，请指定字段名称映射的逗号分隔列表。

logDirectory

目录路径

${server.output.dir}/logs

可以使用此属性来设置所有日志文件（不包括 console.log 文件，但包括 FFDC）的目录。 缺省情况下，logDirectory 设置为 LOG_DIR 环境变量。 缺省 LOG_DIR 环境变量路径为 WLP_OUTPUT_DIR/serverName/logs。 避免麻烦：请使用 LOG_DIR 环境变量或 com.ibm.ws.logging.log.directory 属性，但启动之后尝试动态更改配置的情况除外。

maxFfdc年龄

具有分钟精度的时间段

-1

FFDC 文件被删除之前的最长期望存在时间。 请指定一个正整数，后跟一个时间单位，可以是天 (d)，小时 (h) 或分钟 (m)。 例如，将 2 天指定为 2d。 可以将多个值包括在单个条目中。 例如，2d6h 等于 2 天 6 小时。 每天午夜达到最长存在时间的任何 FFDC 文件都会被删除。 缺省情况下，不会基于文件存在时间删除 FFDC 文件。 指定后跟时间单位的正整数，时间单位可以是小时 (h) 或分钟 (m)。 例如，以 30m 的形式指定 30 分钟。 可以将多个值包括在单个条目中。 例如，1h30m 相当于 90 分钟。

maxFileSize

int
最小值: 0

20

日志文件在轮转之前可以达到的最大大小（以 MB 计）。 Liberty 运行时仅执行基于大小的日志轮转。 要禁用此属性，请将值设置为 0。 最大文件大小是近似值。 缺省情况下，此值为 20。 注意：maxFileSize 不适用于 console.log 文件。

maxFiles

int
最小值: 0

2

移除最早的日志文件之前保留的最大日志文件数；值为 0 意味着无限制。 如果强制最大文件大小存在，那么此设置用于确定每个日志文件保留的字节数。 此设置也适用于对特定日期发生的异常进行汇总的异常日志数。 因此，如果此数目为 10，那么 ffdc/directory 中可以具有 10 个消息日志、10 个跟踪日志和 10 个异常日志。 缺省情况下，此值为 2。 注意：maxFiles 不适用于 console.log 文件。

messageFileName

字符串

messages.log

消息输出将写入的文件的名称（相对于配置的日志目录）。 缺省值为 messages.log。 此文件始终存在，并且包含 INFO 消息和其他（AUDIT、WARNING、ERROR 和 FAILURE）消息以及 System.out 和 System.err。 此日志还包含时间戳记和发出消息的线程标识。 如果日志文件将轮转，那么较早日志文件的名称具有 messages_timestamp.log 格式。 避免麻烦：请使用 com.ibm.ws.logging.message.file.name 属性，但启动之后尝试动态更改配置的情况除外。

messageFormat

  • JSON

  • SIMPLE

  • 待定

SIMPLE

messages.log 文件的必需格式。 有效值为 SIMPLE 或 JSON 格式。 缺省情况下，messageFormat 设置为 WLP_LOGGING_MESSAGE_FORMAT 环境变量（如果已设置）或者 SIMPLE。 避免麻烦：请使用 WLP_LOGGING_MESSAGE_FORMAT 环境变量或 com.ibm.ws.logging.message.format 属性，但启动之后尝试动态更改配置的情况除外。
JSON
使用 JSON 日志记录格式。
SIMPLE
使用简单日志记录格式。
TBASIC
使用 tbasic 日志记录格式。

messageSource

字符串

消息

路由至 messages.log 文件的源的逗号分隔列表。 仅当 messageFormat=json 时，此属性才适用。 有效值为 message、trace、accessLog、ffdc 和 audit。 缺省情况下，messageSource 设置为环境变量 WLP_LOGGING_MESSAGE_SOURCE（如果已设置此环境变量）或 message。 注意：要使用审计源，请启用 Liberty 审计功能部件。 通过为 httpEndpoint 设置 accessLogging 元素来启用访问日志。 避免麻烦：请使用 WLP_LOGGING_MESSAGE_SOURCE 环境变量或 com.ibm.ws.logging.message.source 属性，但启动之后尝试动态更改配置的情况除外。

rolloverInterval

具有分钟精度的时间段

-1

日志回滚之间的时间间隔，以分钟为单位（如果未指定时间单位）。 请指定一个正整数，后跟一个时间单位，可以是天 (d)，小时 (h) 或分钟 (m)。 例如，将 5 小时指定为 5h。 可以将多个值包括在单个条目中。 例如，1d5h 相当于 1 天零 5 个小时。 如果指定了 rolloverStartTime，那么 rolloverInterval 的缺省值为 1 天。 如果同时未指定 rolloverInterval 和 rolloverStartTime，那么将禁用基于时间的日志回滚。 注：rolloverInterval 不适用于 console.log 文件。 指定后跟时间单位的正整数，时间单位可以是小时 (h) 或分钟 (m)。 例如，以 30m 的形式指定 30 分钟。 可以将多个值包括在单个条目中。 例如，1h30m 相当于 90 分钟。

rolloverStart时间

字符串

第一次回滚的日志的调度时间。 回滚时间间隔持续时间从回滚开始时间开始。 有效值遵循 24 小时 ISO-8601 的日期时间格式 HH:MM，其中 00:00 表示午夜。 需要填充零。 如果指定了 rolloverInterval，那么 rolloverStartTime 的缺省值为 00:00（午夜）。 如果同时未指定 rolloverInterval 和 rolloverStartTime，那么将禁用基于时间的日志回滚。 注意：rolloverStartTime 不适用于 console.log 文件。

stackTraceSingleEntry

布尔型

否

将写入 System.out/System.err 的堆栈跟踪作为单个事件处理在日志中。

suppressSensitiveTrace

布尔型

否

当此属性设置为 true 时，会防止在日志和跟踪文件中暴露潜在的敏感信息。 缺省值为 false。 避免麻烦：除非在启动后尝试动态更改配置，否则请使用 com.ibm.ws.logging.suppress.sensitive.trace 属性。

traceFileName

trace.log

跟踪输出将写入的文件的名称（相对于配置的日志目录）。 缺省值为 trace.log。 仅当设置了 traceSpecification（包括记录级别低于 INFO）时，才会创建 trace.log 文件。 stdout 将识别为特殊值，它会导致将跟踪定向至原始标准输出流。 避免麻烦：请使用 com.ibm.ws.logging.trace.file.name 属性，但启动之后尝试动态更改配置的情况除外。

使服务器关闭活动的日志文件，并在一天中的指定时间开始新日志文件。 指定 fileSwitchTime 的值时，将启用文件切换，否则将禁用该功能。

outOfSpaceAction

  • PurgeOld

  • StopLogging

  • StopServer

StopLogging

指定用于保存记录的文件系统的可用空间不足时要执行的操作。 如果将 outOfSpaceAction 设置为“StopLogging”，那么当记录无法写入到磁盘时服务器将停止日志记录。 当此属性设置为“PurgeOld”时，服务器将尝试从二进制日志存储库删除最早的记录，以便为新记录腾出空间。 如果此属性设置为“StopServer”，那么当无法写入记录时二进制日志将停止服务器。

最小值: 0

50

0
