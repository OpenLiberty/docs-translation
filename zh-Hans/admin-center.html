使用 Admin Center 管理 Open Liberty

Admin Center 是 Open Liberty 服务器的基于 Web 的图形用户界面 (GUI)。 您可以使用管理中心来管理服务器配置， OpenID Connect 许可权和 Java 批处理作业。

设置管理中心

  1. 如果在运行时中尚未安装 Admin Center 功能部件，请通过从 path_to_liberty/wlp/bin 目录运行 featureUtility installFeature 命令来安装该功能部件，如以下示例中所示。

    featureUtility installFeature adminCenter-1.0

  2. 在 server.xml 文件中启用 管理中心功能 ，并为读者或管理员角色 配置管理角色 。

  3. Admin Center 只能通过 HTTPS 访问，您必须指定 HTTPS 端口并配置密钥库以保护包含服务器认证凭证的 keystore 文件。 您可以使用以下示例 server.xml 文件来设置管理中心的测试实例。

此示例启用 adminCenter-1.0 功能部件并配置用于定义 reader-role 和 administrator-role 管理角色中的用户的 基本用户注册表 。 它还在 httpEndpoint 元素中指定 HTTPS 端口，并在 keyStore 元素中指定缺省密钥库和密码引用。

<server description="Admin Center example server">
   <featureManager>
      <feature>adminCenter-1.0</feature>
   </featureManager>
   <basicRegistry id="basic">
      <user name="admin" password="adminpwd" />
      <user name="reader" password="readerpwd" />
   </basicRegistry>
   <administrator-role>
      <user>admin</user>
   </administrator-role>
   <reader-role>
      <user>reader</user>
   </reader-role>
   <keyStore id="defaultKeyStore" password="Liberty" />
   <httpEndpoint id="defaultHttpEndpoint"
                 host="*"
                 httpPort="9080"
                 httpsPort="9443" />
</server>

登录到管理中心

在访问 Admin Center GUI 之前，请确保服务器正在运行。 在 Web 浏览器中，转至位于以下 URL 的 Admin Center 登录页面。

https://host_name:port_number/adminCenter

对于 host_name 值，指定运行 Open Liberty 服务器的计算机的 IP 地址或域名服务器 (DNS) 主机名。 如果浏览器与服务器在同一计算机上运行，那么可以将 localhost 指定为主机名。 对于 port_number 值，请从 server.xml 文件中的 httpEndpoint 元素指定 httpsPort 值。

例如，如果您的浏览器与服务器在同一台计算机上运行，并且您使用先前的示例 server.xml 配置，那么可从以下 URL 访问 Admin Center。

https://localhost:9443/adminCenter

仅当访问权直接来自浏览器时，才支持访问 Admin Center。 不支持通过 HTTP 代理服务器进行访问。

如果浏览器提示您确认连接可信，请指定异常或以其他方式启用连接以继续到 Admin Center。 要登录到 Admin Center ，请指定管理角色配置中的用户名和密码。 例如，如果使用先前提供的示例 server.xml 配置，请指定 admin 作为用户名，指定 adminpwd 作为密码。

使用 "服务器配置" 工具管理和测试服务器配置

您可以使用 服务器配置 工具来查看和编辑 Open Liberty 服务器配置文件以及测试与不同资源的服务器连接。 要编辑配置文件或测试连接，您必须处于 administrator-role 管理角色。

在您可以编辑配置文件之前，该工具会提示您将 remoteFileAccess 元素添加到 server.xml 文件。 您可以从浏览器窗口复制示例元素，也可以手动对其进行配置。 如果不添加此元素，那么将以只读方式显示配置文件。

要启用对配置文件的写访问权，请将以下代码添加到 server.xml 文件。

<remoteFileAccess>
   <writeDir>${server.config.dir}</writeDir>
</remoteFileAccess>

服务器配置 工具以两种方式显示配置文件 (例如 server.xml 文件): 设计 和 源。 可以从工具中的浏览器选项卡访问其中每种方式。 您可以通过单击 设置 图标来定制这些方式。 例如，您可以在 设计 方式下添加或除去参数描述，也可以在 源 方式下添加或除去行号。

设计 方式通过将图形控件与内联文档配合使用来显示配置文件的内容。 在此方式下，选择配置中的元素以查看已启用的功能部件和参数设置。

源 方式允许直接访问以编辑文件文本，并提供文件元素的工具提示描述。 将鼠标悬停在元素和参数上以查看其文档。 可按 Ctrl + Space 键以使用内容辅助，这可以帮助您添加新元素、参数和值。 但是， 源 方式编辑器不会验证您指定的配置值。 您在 源 方式下进行的更改将应用于正在运行的服务器。 指定无效的配置值可能会产生错误或导致服务器失败。

您可以使用服务器配置工具来测试与 Java 数据库连接 (JDBC) ， Java EE 连接器体系结构 (JCA) ， Java 消息服务 (JMS) 和 Cloudant 资源的连接。

    以下截屏显示 " 连接测试 " 页面和认证方法。

Admin Center 使用 REST API 来验证连接。 有关更多信息，请参阅 验证服务器连接。

使用浏览工具监视和管理资源

您可以使用 浏览 工具来监视和管理 Open Liberty 资源。 此工具提供用于查看有关服务器及其应用程序的信息的选项。 您还可以使用它来停止，启动或重新启动资源。

要在 浏览 工具中更及时地更新服务器和应用程序状态，请将 Web 套接字 功能部件添加到服务器配置。 此功能部件提供拓扑的实时视图，而不考虑大小。 如果没有 "Web 套接字" 功能，那么 Admin Center 会定期且频繁地轮询更改。

您可以从导航栏中选择 应用程序 视图，以监视，停止，启动或重新启动正在服务器上运行的应用程序。

导航栏上的 " 监视器 " 视图以图形方式在图表中显示服务器和系统度量。 您可以使用 "监视器" 视图来跟踪堆内存，已装入的类，活动 Java 虚拟机 (JVM) 线程和中央处理器 (CPU) 使用情况。

以下截屏显示 浏览 工具的 " 监视器 " 视图。

您可以通过选择 编辑 图标并从菜单中选择选项，向 " 监视器 " 视图添加更多资源度量。 启用 性能监视 功能后，此菜单中提供了更多资源度量值。 根据为服务器配置的资源，以下资源指标可能可用:

  • 活动会话数

  • 活动 Liberty 线程

  • 平均响应时间

  • 平均等待时间

  • 已用连接数

使用 Java 批处理工具管理 Java 批处理作业

如果配置 批处理管理 功能，那么可以访问 Java 批处理 工具。 通过此工具，您可以查看 Java 批处理作业的进度和状态，管理其实例以及查看其日志文件。

以下截屏显示了 Java 批处理 工具。

每个批处理作业都有一个 操作 图标 (可以选择停止，重新启动或清除作业) 和一个 查看日志 图标 (可以选择查看作业日志)。 要查看批处理作业的详细信息和度量值，请将鼠标悬停在 批处理作业名称 上，然后从工具提示菜单中选择一个选项。

如果批处理作业或作业日志位于远程服务器上，请在每个远程服务器上 配置跨源区域共享 (CORS) 。 CORS 使管理中心能够从远程服务器请求作业信息。

"批处理管理" 功能需要定制授权才能查看和管理批处理作业。 要使用 Java 批处理工具，除了访问管理中心所需的读者或管理员管理角色外，还必须配置 com.ibm.ws.batch 定制授权角色。 以下 server.xml 文件示例显示授予管理员管理角色和 batchAdmin 定制授权角色的 wanda 用户的配置。

  <administrator-role>
      <user>wanda</user>
  </administrator-role>

  <authorization-roles id="com.ibm.ws.batch">
      <security-role name="batchAdmin">
          <user name="wanda"/>
      </security-role>
  </authorization-roles>

通过此配置，用户有权查看和管理任何已配置的 Java 批处理作业。

com.ibm.ws.batch 定制授权角色还可以与读者管理角色组合。 此组合仍然允许对 Java 批处理工具的完全访问权，但提供对其他管理中心资源的只读访问权。

但是，如果未配置定制批处理授权角色，那么即使具有管理员管理角色的用户也无法查看或管理 Java 批处理作业。
